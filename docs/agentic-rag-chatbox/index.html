从 Chatbox 到 Agentic RAG

一个 NotebookLM-like 问答库，背后如何做意图识别、检索、工具调用、证据组织、答案规划与可解释输出？

Course Goal

这不是展示页，而是一套互动课件

  • 前测：先暴露你对 RAG / Agent 的直觉误区
  • 互动图：把 Chatbox 后端拆成可点击模块
  • 练习：自己判断 route，并生成 Tool Plan
  • 后测：确认你能产出自己的 Chatbox 框架

Objectives

学完你要能产出什么？

  • 画出 Chatbox 后端链路：Input → Intent → Retrieval → Evidence → Planner → Verifier → UI
  • 定义 Intent Route 表：definition / process_operation / evidence_lookup / comparison / table_lookup
  • 写出 Tool Plan：检索、读上下文、构造证据、答案规划、事实校验
  • 设计 Reasoning Summary：展示可审计执行轨迹，而不是 raw chain-of-thought
  • 给当前 QT-wiki 项目列出下一步升级路线

Case

当前项目真实链路

/api/session/query\n→ 读取 Tool/output/parsed/{document_id}.json\n→ runtime 构建 SectionChunk\n→ 判断 route，例如 process_operation\n→ HybridRetriever: RuleSectionRetriever + FullTextRetriever\n→ route-aware rerank\n→ AnswerEvidencePackage\n→ citation validation\n→ answer_run / evidence_preview / self_check
已做：文件型 canonical store + runtime chunk + lexical hybrid retrieval + citation validation。
未做：persistent chunk index、embedding cache、runtime VectorRetriever、Answer Planner、Claim Verifier。
