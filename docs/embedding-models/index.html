Core

从One-Hot到稠密向量

已经知道：计算机只能处理数字，所以文本必须转换成数值表示。

但是问题是：最简单的One-Hot编码把每个词表示为一个N维向量（N=词汇量），除了一个位置是1其余全0。"猫"和"狗"的One-Hot向量之间没有任何数学联系。

所以这一段要解决：稠密向量（Dense Embedding）用较少维度（如768维）的连续值，让语义相近的词在向量空间中靠得更近。

One-Hot编码

猫 → [1,0,0,0,...]（5万维）
狗 → [0,1,0,0,...]（5万维）
cos(猫,狗) = 0 —— 完全不相似
无法捕捉语义关系

稠密向量 (Embedding)

猫 → [0.12, -0.34, 0.87, ...] (768维)
狗 → [0.15, -0.28, 0.82, ...] (768维)
cos(猫,狗) ≈ 0.85 —— 高度相似！
语义相近 → 向量方向相近

Core

余弦相似度：量化语义距离

在嵌入空间中，我们用余弦相似度来衡量两个向量"方向"的接近程度：

cos(A, B) = (A · B) / (||A|| × ||B||)

取值范围 [-1, 1]，越接近1表示语义越相似

✅ 余弦相似度高

猫 vs 狗 → 0.85
Python vs JavaScript → 0.78
深度学习 vs 神经网络 → 0.92

❌ 余弦相似度低

猫 vs 汽车 → 0.12
Python vs 篮球 → 0.05
深度学习 vs 烹饪 → 0.08

Core

降维可视化：看到高维空间

嵌入向量通常是256-3072维的，人类无法直接"看到"。降维算法将它们映射到2D/3D：

PCA（主成分分析）

线性方法，保留全局结构
速度：快
适用：快速探索、保留距离比例
局限：非线性关系可能丢失

t-SNE（t分布随机邻域嵌入）

非线性方法，保留局部结构
速度：慢
适用：发现聚类、可视化语义群
局限：全局距离可能失真

示意：2D降维后的词向量空间。同色点表示语义相近的词聚类在一起。

Worked Example

嵌入在实战中的应用

嵌入模型是现代NLP的基础设施。以下是三个核心应用场景：

1. RAG检索

用户查询 → 嵌入模型 → 查询向量 → 在ChromaDB中搜索最相近的文档块向量 → 返回top-k结果。嵌入质量直接决定检索召回率。

2. 语义搜索

传统关键词搜索："猫粮"找不到"宠物食品"。语义搜索通过嵌入，将查询和文档映射到同一空间，cos("猫粮", "宠物食品") ≈ 0.8 → 成功匹配。

3. 文本分类 & 聚类

将文本嵌入后，相似主题的文档在向量空间中自然聚集。可以用K-Means或DBSCAN进行无监督聚类，或用嵌入向量作为分类器的输入特征。

# Python示例：计算两个句子的余弦相似度 from openai import OpenAI import numpy as np client = OpenAI() def get_embedding(text): resp = client.embeddings.create( model="text-embedding-3-small", input=text ) return resp.data[0].embedding a = get_embedding("猫吃鱼") b = get_embedding("狗啃骨头") c = get_embedding("今天天气不错") cos_ab = np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b)) cos_ac = np.dot(a, c) / (np.linalg.norm(a) * np.linalg.norm(c)) print(f"猫吃鱼 vs 狗啃骨头: {cos_ab:.3f}") # → ~0.75 print(f"猫吃鱼 vs 今天天气: {cos_ac:.3f}") # → ~0.15

Summary

小结：语义空间的奥秘

我们回答了一开始的问题：计算机通过嵌入模型把词语映射到高维向量空间，语义相近的词在这个空间中靠得更近。余弦相似度量化了这种语义距离——"猫"和"狗"的相似度远高于"猫"和"汽车"。

01 从稀疏到稠密：One-Hot维度=词汇量，稀疏且无语义；Embedding维度=256-3072，稠密且语义丰富。这是NLP史上最重要的表征革命。
02 余弦相似度 = A·B / (||A||×||B||)，取值范围[-1,1]。越接近1语义越近，0表示正交（不相关），-1表示语义相反。
03 模型选型指南：通用选text-embedding-3-small，中文选bge-large-zh，长文本多语言选jina，极致精度选text-embedding-3-large。
04 嵌入无处不在：RAG检索、语义搜索、文本聚类、推荐系统——嵌入是现代AI应用的基础设施。
